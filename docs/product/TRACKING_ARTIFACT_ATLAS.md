# NodeVideo Tracking Artifact Atlas

## Product split

NodeVideo now treats the attached NodeSlide pattern as three related media products:

1. **Artifact Atlas** — searchable before/after proof for each tracking and reframing archetype.
2. **Detector Arena** — same common artifact contract, compared across cheap automatic detectors,
   explicit target seeding, and later specialist models.
3. **Ultra Showcase** — the downloadable compilation assembled only from passing, rights-cleared
   fixture artifacts.

## Structural directions considered

| Direction | Strength | Rejected weakness |
| --- | --- | --- |
| Editor mode inside Creator | Immediate editing context | Overloads the active campaign with reference media |
| Standalone marketing gallery | Strong visual impact | Loses detector, receipt, and harness evidence |
| Proof-backed Atlas workspace | Artifact-forward and inspectable | Requires a dedicated route and responsive guide rail |

The implemented direction is the proof-backed Atlas workspace. It keeps the creator app focused
while using the same artifact-first, persistent-right-rail constitution as NodeVideo, NodeSlide,
NodeRoom, and Parity Studio.

## Common contract

```text
Creator intent
→ domain-pack selection
→ specialist detection or explicit target seed
→ DetectionTrack
→ ActionEnvelope
→ ReframePlan
→ ReframeCritic
→ EditPlan proposal
→ approval
→ preview/export
→ receipt
```

Group, object, animal, and sports packs never own a separate renderer or write path. They provide
target semantics, detector routing, envelope policy, fixtures, and evaluation thresholds.

## Current proof boundary

- Eight Creative Commons Attribution YouTube fixtures are downloaded into ignored QA evidence.
- Six fixtures use local YOLO11n COCO discovery.
- Coffee/product and basketball use an explicit first-frame seed with local OpenCV template
  tracking because the cheap detector did not reliably discover their intended target.
- Each case emits source attribution, source hash, detector/tracker identity, latency, coverage,
  limitations, output hashes, before/after images, comparison video, analysis video, and receipt.
- The pass applies to the named fixture and common contract. It does not establish universal
  identity continuity, event understanding, or held-out creator preference.

## Promotion ladder

```text
Contract
→ rights-cleared fixture
→ adversarial fixture
→ held-out creator media
→ creator preference
→ export/reopen parity
→ production pack
```

No detector becomes the default merely because its fixture passes. The Atlas makes both the
automation level and claim boundary visible.
