## Release blockers to prove, not assumed defects

1. The clean `/` route visibly distinguishes public synthetic demo from local-only user media.
2. Loading/running the public synthetic demo performs zero cross-origin egress and never implies a real worker/model/render.
3. Stage, trace, and artifact surfaces agree on IDs, status, parentage, and provenance.
4. A proposal cannot mutate the recipe before acceptance; double acceptance creates exactly one version.
5. Reload recovers the local checkpoint honestly without persisting a usable object URL.
6. Error/partial/unknown states remain distinct and preserve valid prior artifacts.
7. Lint, typecheck, unit, build, and Playwright gates pass on the same current tree.
8. Responsive and supported-theme screenshots receive a vision-capable review before B9 is scored.

## Privacy regression list

Every future pass must re-check these even if there are no open findings:

- no HTTP(S) egress from the synthetic path except same-origin app assets;
- no user video, derived frame/audio, object URL, absolute local path, or credential in Git/CI/evidence/logs/traces/checkpoints;
- no synthetic result mislabeled as browser-local worker output or LIVE model output;
- no persisted session-only object URL after reload;
- no private filename in shared evidence unless explicitly approved and necessary.
