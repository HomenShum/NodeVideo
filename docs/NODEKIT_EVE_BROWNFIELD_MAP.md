# NodeKit brownfield map for Eve

Status: **mapping-only**. This change describes the existing NodeVideo agent
application without moving files, replacing Eve, or changing runtime behavior.

## Canonical application surface

| NodeAgent concern | Existing NodeVideo authority |
| --- | --- |
| Application manifest | `nodeagent.yaml` |
| Authored agent tree | `apps/eve-agent/agent/` |
| Runtime | Eve `0.24.4` |
| Model route | Eve's Vercel AI Gateway route for `openai/gpt-5.4-mini` |
| Backend adapter | Typed generation and evaluation control APIs in `apps/eve-agent/agent/lib/control-api.ts` |
| Capability pack | `packs/song-conditioned-auto-edit/manifest.json`, mapped by adjacent `pack.yaml` |
| Local isolation gate | `apps/eve-agent/scripts/verify-isolation.mjs` |
| Live Eve evals | `apps/eve-agent/evals/` |

The legacy `manifest.json` remains NodeVideo's application-owned pack contract.
The adjacent `pack.yaml` is canonical NodeAgent metadata over those existing
tools, artifacts, prompts, fixture, and replay eval. It is not a second worker
implementation.

## Runtime and credential boundary

`runtime.engine` is `eve`. The manifest does not claim a NodeAgent-native
runtime, Pi adapter, direct Convex execution, or a shared local/eval/production
engine. Eve routes the authored string model through Vercel AI Gateway. Local or
non-Vercel live use needs `AI_GATEWAY_API_KEY`; a linked Vercel deployment may
use project OIDC instead. Generation and evaluation control URLs and bearer
tokens remain separately configured, as required by the existing adapter.

The root deterministic replay and proof commands validate NodeVideo's fixed
workers and public artifacts. They do not execute the Eve conversation runtime.
The Eve live eval command requires model credentials and configured control
endpoints. Production browser access remains fail-closed behind
`placeholderAuth()` until the application authenticator is implemented. No live
eval, deployment, or production-parity claim follows from this mapping.

## Validation

Run the repository-owned structural map check:

```powershell
npm run nodeagent:validate
```

Run the existing Eve package gate with Node 24:

```powershell
npm --prefix apps/eve-agent ci
npm --prefix apps/eve-agent run check
```

The live eval is an explicit credentialed gate, not part of the no-key check:

```powershell
npm --prefix apps/eve-agent run eval
```

## Compiled artifact boundary

The committed `.nodeagent/` directory was generated by the unreleased NodeKit
`0.2.0` compiler at `node-platform` commit `f4ac7c6` on
[draft pull request #4](https://github.com/HomenShum/node-platform/pull/4).
That compiler adds repository-contained discovery for a declared
`authoring.directory` and referenced pack roots. This dependency is not yet
available from npm.

The generated discovery binds all 63 tracked files under
`apps/eve-agent/agent`, the canonical pack metadata, both top-level eval
bindings, and the existing files in NodeKit's standard discovery roots. The
result is 217 authored files under config hash
`e1ee1155566d2d66e9cc007cb6d1755af6decf7929c86b1eeb4cfb04bbb2fa90`.
Diagnostics contain no errors or warnings, and the resolved artifact contains
credential names only, never credential values.

NodeKit classifies both canonical `subagents/**/agent.yaml` definitions and
adapter-authored `subagents/**/agent.ts` definitions, so the resolved summary
names all three existing Eve subagents without requiring a duplicate YAML tree.

Reproduce compilation from this repository with the draft compiler checkout:

```powershell
$env:NODE_PLATFORM_ROOT = '<node-platform checkout at f4ac7c6>'
node "$env:NODE_PLATFORM_ROOT/src/cli.mjs" compile --repo-root .
```

`npm run nodeagent:validate` independently re-hashes every discovered file,
checks that the entire Eve authoring tree is present, and verifies the manifest
digest and config-hash linkage. The compiled definition is metadata; the Eve
runtime does not consume it yet. It does not prove a shared benchmark, local,
or production execution path, NodeAgent-native events, shadow parity, a live
eval, or a deployed authenticated Eve service.
