# Recover from Generic Dashboard Fallback

Trigger when an intended artifact workspace becomes a card wall, form/dashboard, proof console, or vertically stacked mobile page.

1. Freeze the current screen as an anti-reference.
2. State the primary user job and the one artifact that proves progress.
3. Split first arrival from active operation.
4. Assign one dominant artifact surface and one adjacent agent/review surface.
5. Turn supporting stages into compact navigation or contextual drawers.
6. Move backstage complexity into a Run Inspector or Settings.
7. Redesign mobile as mutually exclusive surface modes.
8. Render at least three directions against the same fixture before implementation.
9. Gate on topology and the real product journey, not build success alone.
