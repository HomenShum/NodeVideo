# CreatorBench methodology

## Workflow admissibility

CreatorBench does not multiply every source by every workflow. Each source declares a corpus tier and the workflows for which its duration, audio, assets, and annotations make it a valid test input. A six-second silent motion clip can test tracking and reframing, but it cannot honestly test filler removal, quote extraction, choreography synchronization, reference imitation, or a founder-launch narrative.

The benchmark therefore reports both the source count and the number of workflow-admissible instances. Missing source tiers are coverage gaps, not failures and not synthetic denominators. CreatorBench v1 requires dedicated tiers for short motion, long-form speech, multi-take performance, permitted reference pairs, multi-asset launch videos, and adversarial or corrupt inputs before it can report all eight workflow families as represented.

## Claim boundary

The universal contract means any creator request can enter the same orchestration and artifact protocol. It does not guarantee completion. Performance claims are derived only from frozen, workflow-admissible instances and separately report automatic, assisted, review, abstention, unsupported, technical-failure, and silent-failure outcomes.
